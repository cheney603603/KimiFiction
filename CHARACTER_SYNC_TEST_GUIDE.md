## 测试场景

### 场景1: 执行阶段自动同步（主要场景）

**步骤**：
1. 打开浏览器，访问 `http://localhost:5173`
2. 登录或注册账号
3. 创建一个新小说或选择已有小说
4. 点击"进入工作流"或导航到工作流页面
5. 切换到"角色设计"阶段
6. 在输入框中输入提示词（例如：设计3个玄幻小说的主角和配角）
7. 点击"执行角色设计"按钮
8. 等待执行完成（查看右侧执行日志）

**预期结果**：
- ✅ 执行日志显示"成功同步 X 个角色到数据库"
- ✅ 左侧"阶段成果"中显示角色数量和角色列表
- ✅ 切换到"角色管理"页面，能看到新生成的角色
- ✅ 角色卡片显示详细信息（年龄、性别、MBTI等）

### 场景2: 手动标记完成同步

**步骤**：
1. 在工作流页面，切换到"角色设计"阶段
2. 点击"执行角色设计"生成角色数据
3. 等待执行完成
4. 在左侧"阶段成果"中，点击"角色设计"展开详情
5. 点击"标记完成"按钮

**预期结果**：
- ✅ 角色设计阶段显示为已完成（绿色勾选）
- ✅ 切换到"角色管理"页面，能看到角色
- ✅ 如果之前已经有角色，新角色会添加到列表中

### 场景3: 验证角色数据完整性

**步骤**：
1. 切换到"角色管理"页面
2. 查看角色卡片，检查以下字段：
   - 姓名（name）
   - 角色类型（role_type: protagonist/antagonist/supporting）
   - 年龄（age）
   - 性别（gender）
   - 外貌（appearance）
   - 性格（personality）
   - MBTI类型（mbti）
   - 背景（background）
   - 目标（goals）
   - 恐惧（fears）
   - 技能（skills）
   - 关系（relationships）

**预期结果**：
- ✅ 所有字段都有数据或合理默认值
- ✅ 角色卡片布局美观，信息清晰

## 常见问题排查

### 问题1: 执行后角色管理页面看不到角色

**检查项**：
1. 查看后端日志，是否有"成功同步 X 个角色到数据库"
2. 运行诊断脚本，检查数据库是否有数据
3. 刷新角色管理页面（Ctrl+F5 强制刷新）

**解决方案**：
- 如果日志显示同步成功但页面看不到，刷新页面
- 如果日志没有同步消息，检查后端是否已重启应用修复

### 问题2: 执行日志正常但没有角色数据

**检查项**：
1. 查看 LLM 返回的原始响应
2. 检查 JSON 解析是否成功
3. 查看 Agent 的 parse_result 方法

**解决方案**：
- 可能是 LLM 返回格式不符合预期
- 尝试重新执行，或调整提示词

### 问题3: 角色数据不完整（缺少某些字段）

**检查项**：
1. 查看 Agent 返回的完整角色数据
2. 检查 profile 字段的结构

**解决方案**：
- 这是正常的，LLM 可能不会生成所有字段
- 可以在角色管理页面手动编辑补充
