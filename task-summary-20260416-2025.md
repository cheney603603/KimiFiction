# 任务总结 - 八维 LLM Rubric 评测系统集成

## 目标
将独立的 `evaluate_novels_llm.py` 评测脚本集成到 KimiFiction 前后端系统。

## 完成内容

### 1. 后端 API 端点 (`training.py` +393 行)

新增三个端点：

**`GET /training/evaluation/references`**
- 列出 `reference/` 目录下所有 .txt 小说文件
- 返回: filename, name, author, size_kb, char_count, word_count

**`POST /training/evaluation/run`**
- 核心评测端点，接受 `text` + `eval_type` 参数
- 加载 `evaluate_rules_llm.csv` 规则文件
- 调用 DeepSeek API 按维度批量判断 88 条规则 Yes/No
- 计算八维度加权得分，返回完整评测结果
- 自动保存到 `training_runs/eval_*.json`

**`GET /training/evaluation/results`**
- 列出最近 20 条评测历史记录

### 2. 前端页面 (`Evaluation.tsx`, 17KB)

- **Tab 1: 选择小说** — 左侧列出 reference 目录小说，右侧文本预览 + 评测控制
- **Tab 2: 历史记录** — 表格展示历史评测结果
- **评测结果展示**: 总分 + 等级 badge + 八维度得分条形图 + 可展开的 88 条规则详情
- 支持 LLM / 关键词两种评测模式切换

### 3. 路由与导航

- `App.tsx`: 添加 `/evaluation` 路由
- `Layout.tsx`: 导航栏新增"八维评测"入口

### 4. Reference 文件读取 (`export.py`)

- 新增 `GET /export/reference/{filename}` 端点，供前端加载小说文本
