# 快速测试指南

## 测试1：角色同步

- 后端服务运行在 `http://localhost:8080`
- 前端服务运行在 `http://localhost:5173`
- 小说ID为1

- ✅ 背景：背景故事

- ✅ 人际关系：表格

1. 在角色详情中点击"编辑"按钮
2. 修改各个字段
3. 点击"保存更改"

预期结果：
- 修改成功

- 刷新页面后数据保持

## 测试2：章节写作和阅读

### 前置条件
- 后端服务运行在 `http://localhost:8080`
- 前端服务运行在 `http://localhost:5173`
- 小说ID为1
- LLM配置已设置

### 测试步骤

#### 1. 生成章节内容

**方式A：通过章节写作页面**
1. 访问：`http://localhost:5173/novel/1/write`
2. 选择章节号（例如第1章）
3. 设置timeout参数（可选，默认15分钟）
4. 点击"生成章节"按钮
5. 等待AI生成完成

**方式B：通过工作流**
1. 访问：`http://localhost:5173/novel/1/workflow/new`
2. 进入章节写作阶段
3. 设置章节号和写作参数
4. 点击"执行"按钮

#### 2. 查看生成的内容

生成成功后，在章节写作页面应该看到：
- ✅ 绿色成功提示："章节生成成功！您可以查看内容，点击"编辑"按钮修改后再保存到数据库"
- ✅ 章节内容显示在页面中部
- ✅ 可以看到编辑和保存按钮

#### 3. 编辑章节内容（可选）

1. 点击"编辑"按钮
2. 在文本框中修改内容
3. 点击"确认保存"

#### 4. 保存章节到数据库

**方式A：直接保存**
1. 点击"保存到数据库"按钮
2. 看到"章节保存成功！"提示

**方式B：编辑后保存**
1. 编辑完成后点击"确认保存"
2. 看到"章节保存成功！"提示

#### 5. 验证章节保存

**方法A：通过API验证**
```bash
# 查看章节列表
curl http://localhost:8080/api/v1/chapters/novel/1

# 查看第1章内容
curl http://localhost:8080/api/v1/chapters/novel/1/chapter/1
```

预期输出：应该包含生成的章节内容

**方法B：通过数据库验证**
```sql
SELECT id, chapter_number, title, LENGTH(content) as content_length, created_at
FROM chapters
WHERE novel_id = 1
ORDER BY chapter_number;
```

**方式A：通过阅读页面**
1. 访问：`http://localhost:5173/novel/1/read/1` （阅读第1章）
2. 应该看到完整的章节内容
3. 可以使用"上一章"/"下一章"导航
4. 可以点击目录按钮查看章节列表

**方式B：通过小说详情页**

2. 点击"阅读小说"按钮
3. 或者点击章节列表中的某个章节

#### 7. 检查章节阅读功能

在阅读页面应该能够：
- ✅ 看到章节标题和内容

## 常见问题排查

### 问题1：角色设计完成后角色管理页面没有新角色

**检查步骤**：

1. **检查工作流状态**
   ```bash
   curl http://localhost:8080/api/v1/workflow/phase-result/1/character_design
   ```
   - 如果返回 `"data":null`，说明角色设计未完成或失败
   - 查看后端日志，查找错误信息

2. **检查数据库**
   ```sql
   SELECT * FROM characters WHERE novel_id = 1;
   ```
   - 如果只有test角色，说明角色未同步

3. **手动触发角色同步**
   - 重新执行工作流角色设计阶段
   - 或者在角色管理页面手动添加角色

### 问题2：章节生成后看不到内容

**检查步骤**：

1. **检查浏览器控制台**
   - 打开开发者工具（F12）
   - 查看Network标签
   - 找到 `POST /api/v1/workflow/chapter/1` 请求
   - 检查响应中的content字段

2. **检查后端日志**
   - 查看 `[撰写第X章] 开始执行` 日志
   - 查看 `[撰写第X章] 完成` 日志
   - 查找错误信息

3. **检查数据库**
   ```sql
   SELECT * FROM chapters WHERE novel_id = 1 ORDER BY chapter_number;
   ```
   - 如果没有章节，说明未保存

4. **重新生成章节**
   - 在章节写作页面重新生成
   - 或者通过工作流执行章节写作阶段

### 问题3：保存后刷新页面内容消失

**原因**：内容未保存到数据库

**解决方法**：
1. 生成后必须点击"保存到数据库"按钮
2. 检查保存是否成功（查看提示信息）
3. 刷新后重新查询数据库验证

### 问题4：阅读页面显示"加载中..."

**原因**：API请求失败或数据不存在

**解决方法**：
1. 检查章节是否已保存到数据库
2. 检查API端点是否正常：`curl http://localhost:8080/api/v1/chapters/novel/1/chapter/1`
3. 检查后端日志，查看错误信息
