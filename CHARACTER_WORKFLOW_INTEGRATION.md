# 角色工作流集成优化总结

## 问题描述

工作流中生成的角色没有对接到角色管理的角色卡中，导致：
1. 角色设计阶段生成的角色数据无法在角色管理页面查看
2. 角色卡的数据格式和展示格式不够完善
3. 缺少友好的编辑界面

## 解决方案

### 1. 后端数据同步

**文件**: `backend/app/workflow_engine.py`

- **`_sync_characters_to_db()` 方法**: 已存在，角色设计阶段自动调用
  - 检查角色是否已存在
  - 存在则更新，不存在则创建
  - 保存完整的 profile 信息

**角色数据格式** (工作流生成):
```json
{
  "name": "角色名",
  "role_type": "protagonist/antagonist/supporting",
  "profile": {
    "age": 年龄,
    "gender": "性别",
    "appearance": "外貌描述",
    "personality": "性格描述",
    "mbti": "MBTI类型",
    "background": "背景故事",
    "goals": ["目标1", "目标2"],
    "fears": ["恐惧1"],
    "skills": ["技能1", "技能2"],
    "relationships": {
      "角色A": "关系描述"
    }
  },
  "arc_description": "角色成长弧线描述"
}
```

**数据库模型** (`backend/app/models/character.py`):
```python
class Character(Base):
    name: str
    role_type: RoleType
    profile: Dict[str, Any]  # JSON 存储完整人设
    current_status: str      # 当前状态（JSON字符串）
    arc_progress: float      # 角色弧光进度 0-1
    first_appearance: int
    last_appearance: Optional[int]
    appearance_count: int
```

### 2. 前端数据获取和展示

**文件**: `frontend/src/pages/CharacterManager.tsx`

#### 数据获取
- 从工作流 API 获取角色设计结果
- 从数据库获取已保存的角色列表
- 合并两个来源的角色数据

#### 角色卡片展示优化

**卡片显示内容**:
- 角色名称
- 角色类型图标（主角/反派/配角/龙套）
- 基本信息卡片：年龄、性别、MBTI
- 外貌简述（2行截断）
- 角色成长进度条
- 出场章节和出场次数
- 工作流生成的角色标记（Sparkles 图标）
- "保存到数据库"按钮（仅工作流生成的角色）

#### 角色详情展示优化

**详细人设部分**（结构化展示）:
1. **基本信息**: 年龄、性别
2. **外貌**: 完整描述
3. **性格**: 性格特征描述
4. **MBTI类型**: 蓝色标签显示
5. **背景**: 背景故事
6. **目标**: 列表展示
7. **恐惧**: 列表展示
8. **技能**: 标签云展示
9. **人际关系**: 表格展示角色间关系

**统计信息**:
- 首次出场章节
- 出场次数
- 角色成长进度百分比

### 3. 角色编辑功能优化

**改进前**: JSON 文本编辑框，用户需要手动编辑 JSON

**改进后**: 字段化表单编辑，包含：

1. **基本信息组**:
   - 年龄输入框
   - 性别下拉选择

2. **外貌描述**: 多行文本框

3. **性格特征**: 多行文本框

4. **MBTI类型**: 输入框（自动大写，4字符限制）

5. **背景故事**: 多行文本框

6. **目标**: 多行文本框（每行一个目标，自动解析为数组）

7. **恐惧**: 多行文本框（每行一个恐惧，自动解析为数组）

8. **技能**: 输入框（逗号分隔，自动解析为数组）

**保存逻辑**:
- 直接更新 editingCharacter 对象
- 调用 updateMutation.mutate() 保存
- 无需 JSON 解析，用户体验更好

## 使用流程

### 工作流生成角色
1. 在工作流页面进入"角色设计"阶段
2. 等待 AI 生成角色设计
3. 生成完成后，自动同步到数据库

### 查看角色
1. 进入角色管理页面
2. 看到所有角色（包括工作流生成的）
3. 点击卡片查看详细人设

### 编辑角色
1. 在角色详情中点击"编辑"按钮
2. 使用表单编辑各个字段
3. 点击"保存更改"保存修改

### 保存工作流角色
1. 工作流生成的角色显示 Sparkles 图标
2. 点击"保存到数据库"按钮
3. 角色保存到数据库，可在之后编辑

## 数据格式映射

### 工作流 -> 数据库
- `name` → `name`
- `role_type` → `role_type`
- `profile` → `profile` (JSON)
- `arc_description` → 不存储（仅用于工作流）

### 数据库 -> 前端
- `name` → 显示
- `role_type` → 图标和标签
- `profile` → 详细人设展示
- `current_status` → 当前状态展示
- `arc_progress` → 进度条和百分比
- `first_appearance` → 首次出场
- `appearance_count` → 出场次数

## 注意事项

1. **角色同步时机**: 角色设计阶段完成后自动同步
2. **数据一致性**: 工作流和数据库的角色通过 name 匹配
3. **编辑限制**: 工作流生成的角色（id=-1）不能直接编辑，需要先保存到数据库
4. **MBTI 格式**: 前端自动大写，限制4字符
5. **数组字段**: 目标、恐惧、技能支持自动从文本解析为数组
