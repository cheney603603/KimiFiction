# Novel Writer — 中长篇网络小说AI协作写作系统

> **版本**: v2.0 | **更新时间**: 2026-03-30

## 核心理念

这是一个**多Agent协作、多阶段渐进式**的智能小说创作系统。不同于简单的一次性生成，系统通过多个专业Agent分工协作，结合RAG检索、上下文压缩、状态管理和错误恢复等机制，实现高质量、可迭代的长篇小说创作。

### 核心特征

- **多Agent协作**: 专业化分工，职责清晰
- **渐进式创作**: 需求→世界观→角色→冲突伏笔→大纲→细纲→正文
- **RAG增强**: 利用向量检索获取相关背景信息
- **上下文压缩**: 确保关键信息在Token限制内
- **状态持久化**: 自动保存工作进度，断点续传
- **错误恢复**: 自动重试、降级处理
- **反馈迭代**: 支持章节重写和框架调整

## 系统架构

### Agent矩阵

| Agent | 职责 | 输入 | 输出 |
|-------|------|------|------|
| `AnalyzerAgent` | 需求分析 | 用户原始需求 | 类型/风格/卖点 |
| `WorldBuilderAgent` | 世界观构建 | 类型+需求 | 世界观设定 |
| `CharacterDesignerAgent` | 角色设计 | 世界观+故事线 | 角色卡 |
| `PlotDesignerAgent` | 冲突伏笔 | 世界观+角色 | 冲突伏笔设定 |
| `OutlineGeneratorAgent` | 大纲生成 | 设定+角色 | 分卷大纲+细纲 |
| `WriterAgent` | 章节撰写 | 细纲+上下文 | 正文 |
| `ReviewerAgent` | 审核修改 | 章节+反馈 | 修改建议/修订 |
| `MemoryManagerAgent` | 记忆管理 | 章节内容 | 记忆节点 |

## 快速开始

### 1. 创建新项目

```
用户: 我想写一个修仙小说，主角是个废柴逆袭

AI: 
我来帮您规划这个修仙小说项目。首先让我分析一下您的需求：

【初步分析】
- 类型: 东方玄幻/修仙
- 核心卖点: 废柴逆袭、升级流、打脸
- 参考风格: 可参考《凡人修仙传》或《斗破苍穹》

【我的问题】
1. 您希望的大致字数是多少？(50万/100万/150万+)
2. 有没有特别喜欢的设定？(宗门流/散修流/古典仙侠风/现代修真)
3. 主角的性格偏好？(沉稳型/热血型/腹黑型)
```

### 2. 确认设定

系统会依次引导确认：
1. **需求分析** → 作品类型、风格、卖点
2. **世界观构建** → 世界设定、力量体系、势力划分
3. **角色设计** → 主角、配角、反派
4. **冲突伏笔** → 主线冲突、伏笔铺设计划

### 3. 生成大纲

```
AI: 
根据您的需求，我设计了以下大纲框架：

【分卷规划】
- 第1卷「觉醒」: 30章，主角发现特殊体质，入门宗门
- 第2卷「成长」: 50章，快速升级，宗门大比崭露头角
- 第3卷「风云」: 50章，卷入大势力争斗
- ...

是否需要调整？
```

### 4. 开始写作

```
用户: 大纲可以，开始写第一章吧

AI: 
好的，开始撰写第1章...

[生成进度: 50%]
正在检索相关上下文...
[生成进度: 80%]
正在撰写正文...
[生成进度: 100%]

✅ 第1章「废物」撰写完成
- 字数: 3280字
- 关键事件: 主角被测出特殊体质、遭受嘲讽
- 伏笔: 体质来源成谜

【下一步】
- 阅读/修改本章
- 继续撰写第2章
- 调整大纲或设定
```

### 5. 反馈迭代

```
用户: 第1章不错，但主角被打脸的场景可以更激烈一些

AI: 
好的，根据您的反馈修订第1章...

✅ 修订完成
- 修改内容: 增强了被打脸场景的描写
- 字数: 3580字 (+300字)

是否需要继续写第2章？
```

## 上下文管理

### RAG检索

撰写章节时，系统自动检索：
- **世界观设定**: 当前世界的规则、力量体系
- **角色状态**: 出场角色的当前状态、成长弧
- **伏笔提示**: 需要在本文处理的伏笔
- **前文摘要**: 最近3-5章的剧情摘要

### 上下文压缩

当上下文超过Token限制时，自动压缩：
1. **保留优先级**: 世界观 > 角色状态 > 未解伏笔 > 前文
2. **压缩策略**: 低优先级内容截断
3. **信息摘要**: 必要时生成摘要替代原文

## 错误处理

### 自动重试策略

| 错误类型 | 策略 | 最大重试 |
|---------|------|----------|
| 限流 (429) | 指数退避 | 3次 |
| 超时 | 线性退避 | 2次 |
| JSON解析失败 | 备用提取 | 2次 |
| LLM错误 | 指数退避 | 3次 |

### 降级处理

当高级功能不可用时：
- 向量检索 → 关键词匹配
- 上下文压缩 → 简单截断
- RAG → 基础上下文

## 最佳实践

### 1. 需求表达

提供**越详细**的需求，生成结果**越准确**：
- ✅ "想写一个现代都市异能文，主角有读心术，后期反转"
- ⚠️ "写个都市异能小说"
- ❌ "写个小说"

### 2. 反馈技巧

- ✅ 具体指出问题："第3段的对话太生硬"
- ⚠️ 模糊反馈："文笔一般"
- ❌ 无反馈："随便"

### 3. 迭代节奏

建议每写3-5章审核一次，而非写完后一次性审核。

### 4. 备份习惯

重要节点（完成大纲、完成第一卷等）手动创建备份：
```
/备份 [描述]
```

## 常见问题

### Q: 工作流中断了怎么办？
**A**: 系统自动保存状态，重新开始时选择「继续未完成项目」即可。

### Q: 生成的内容不满意怎么办？
**A**: 使用 `/重写` 命令，可指定修改范围（全章/特定段落）。

### Q: 如何调整已确定的设定？
**A**: 使用 `/框架调整` 命令，系统会评估影响范围并提供修订计划。

### Q: 可以导出哪些格式？
**A**: 支持 Markdown、TXT、EPUB（规划中）。
