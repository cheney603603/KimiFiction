# 章节写作上下文修复说明

## 问题描述

在章节写作功能中发现以下问题：
1. 缺少前文摘要、角色设定、世界观等上下文信息
2. 角色数据未同步到数据库，导致章节写作时无法获取角色信息
3. 生成的章节内容无法查看和编辑后再保存到数据库

## 修复内容

### 1. 后端修复 (backend/app/workflow_engine.py)

#### 1.1 添加从数据库加载角色的方法
- 新增 `_load_characters_from_db()` 方法
- 在 `write_chapter()` 调用时自动加载角色数据
- 确保章节写作时能获取到正确的角色设定

#### 1.2 添加同步角色到数据库的方法
- 新增 `_sync_characters_to_db()` 方法
- 在角色设计阶段完成后自动将角色同步到数据库
- 支持创建新角色和更新已存在的角色

#### 1.3 改进章节写作上下文构建
- 增强大纲信息获取逻辑，从 `chapter_outlines` 中提取细纲
- 确保 `outline` 包含 `summary`、`key_points`、`scenes` 等关键信息

### 2. 前端修复 (frontend/src/pages/ChapterWriter.tsx)

#### 2.1 添加数据查询
- 获取角色信息列表
- 获取世界观设定
- 获取前文摘要（从上一章）
- 保留章节细纲查询

#### 2.2 新增编辑和保存功能
- 添加编辑状态管理 (`isEditing`, `editedContent`)
- 添加编辑按钮，点击后切换到编辑模式
- 添加保存按钮，将编辑后的内容保存到数据库
- 添加取消编辑按钮

#### 2.3 改进上下文显示
- 在设置面板中添加"查看上下文"按钮
- 显示世界观设定、相关角色、前文摘要等信息
- 使用不同颜色区分不同类型的上下文信息

#### 2.4 优化用户界面
- 章节标题栏添加编辑/保存按钮
- 编辑模式下使用 textarea 方便用户修改内容
- 保存时显示加载状态
- 生成成功后不自动保存，等待用户确认

## 功能说明

### 章节写作流程

1. **选择章节**：从下拉菜单选择要编写的章节
2. **查看上下文**：点击"查看上下文"查看世界观、角色、前文摘要
3. **查看细纲**：点击"查看细纲"查看章节大纲和关键点
4. **设置参数**：调整文风、环境描写级别、对话占比等
5. **生成内容**：点击"开始生成"按钮，AI根据上下文生成章节
6. **编辑内容**：生成完成后，可以点击"编辑"按钮修改内容
7. **保存章节**：满意后点击"保存到数据库"将章节保存

### 数据流转

```
工作流角色设计 → 存储到工作流状态 → 同步到数据库characters表
                          ↓
                     章节写作时从数据库加载
                          ↓
                      结合世界观、前文摘要
                          ↓
                      构建完整上下文
                          ↓
                      AI生成章节内容
                          ↓
                      用户编辑确认
                          ↓
                      保存到chapters表
```

## 使用说明

### 前提条件

1. 小说已完成角色设计阶段（工作流中）
2. 已有章节细纲（工作流中）
3. 已配置 LLM 服务

### 操作步骤

1. 进入"继续创作"页面 (`/novel/:novelId/write`)
2. 选择要编写的章节编号
3. 查看"查看上下文"确认世界观、角色、前文摘要已加载
4. 查看"查看细纲"确认章节大纲信息
5. 调整"写作设置"中的参数（可选）
6. 点击"开始生成"等待AI生成
7. 生成完成后，如需修改，点击"编辑"按钮
8. 确认无误后，点击"保存到数据库"

## 注意事项

1. **角色同步**：角色设计完成后会自动同步到数据库，如果未同步可重新执行角色设计阶段
2. **前文摘要**：从上一章的 `summary` 字段获取，如果上一章没有摘要则不显示
3. **章节保存**：生成的章节不会自动保存，必须手动点击保存按钮才会写入数据库
4. **章节覆盖**：如果章节已存在，保存时会更新现有章节内容
5. **编辑模式**：编辑时可以修改标题、摘要等内容

## 技术细节

### 后端 API

- `POST /api/v1/workflow/chapter/{novel_id}` - 章节写作
- `GET /api/v1/characters/{novel_id}` - 获取角色列表
- `POST /api/v1/chapters` - 创建章节
- `PUT /api/v1/chapters/{id}` - 更新章节
- `GET /api/v1/workflow/phase/{novel_id}` - 获取工作流阶段结果

### 数据库表

- `characters` - 角色表
  - `profile` (JSON) - 详细人设
  - `current_status` (TEXT/JSON) - 当前状态
  - `role_type` (ENUM) - 角色类型

- `chapters` - 章节表
  - `content` (LONGTEXT) - 章节正文
  - `summary` (TEXT) - 章节摘要
  - `characters_present` (JSON) - 出场角色列表
