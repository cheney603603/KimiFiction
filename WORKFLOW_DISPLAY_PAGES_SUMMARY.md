# 工作流生成结果展示功能

## 概述
本次修改为小说项目的四个主要页面添加了对工作流生成结果的读取和展示功能，使得用户可以方便地查看和编辑工作流各阶段生成的数据。

## 修改内容

### 1. 角色管理页面 (`CharacterManager.tsx`)

**新增功能：**
- 读取工作流阶段 `character_design` 生成的角色数据
- 合并工作流角色和数据库角色，统一展示
- 工作流生成的角色显示紫色星标，与数据库角色区分
- 支持"保存到数据库"功能，将工作流角色持久化
- 新增编辑功能，支持修改角色人设（JSON格式编辑）
- 显示工作流角色数量提示

**主要改进：**
- 角色卡片显示更详细的人设信息
- 支持查看工作流生成的角色详细设定
- 提供便捷的保存和编辑操作

### 2. 大纲编辑页面 (`OutlineEditor.tsx`)

**新增功能：**
- 读取工作流阶段 `outline_draft` 生成的剧情大纲
- 读取工作流阶段 `outline_detail` 生成的章节细纲
- 分别展示剧情大纲和章节细纲两个区域

**剧情大纲展示：**
- 故事概述
- 分卷大纲（可展开/收起）
- 剧情弧光列表

**章节细纲展示：**
- 章节标题和摘要
- 章节要点列表
- 角色出场信息
- 关键事件和伏笔
- 场景设定（地点、时间、目的）
- 每章可展开查看详细内容

**主要改进：**
- 清晰区分工作流数据和已保存数据
- 支持查看完整的章节细纲信息
- 提供章节内容预览

### 3. 继续创作页面 (`ChapterWriter.tsx` - 新建)

**页面功能：**
- 单一章节撰写功能，无需切换阶段
- 选择章节编号（下拉选择）
- 自动获取该章节的细纲信息作为上下文

**写作设置：**
- 文风选择：现代简洁、古风典雅、网络轻松、严肃深沉、幽默诙谐
- 环境描写级别：极简、正常、丰富
- 对话占比：0-100% 可调节
- 写作注意事项：自定义输入

**章节细纲集成：**
- 自动加载对应章节的细纲
- 显示章节摘要和要点
- 提供上下文参考

**内容展示：**
- 实时显示生成的章节内容
- 支持查看字数统计
- 已存在章节可直接查看

### 4. 路由配置 (`App.tsx`)

**新增路由：**
- `/novel/:novelId/write` - 章节创作页面

### 5. 导航更新 (`NovelDetail.tsx`)

**修改项：**
- "继续创作" 菜单项的链接从 `/novel/${id}/workflow` 改为 `/novel/${id}/write`
- 更新描述为"单一章节撰写，快速生成内容"

## 技术实现

### API 调用

使用 `workflowApi.getPhaseResult(novelId, phase)` 获取各阶段数据：
- `character_design` - 角色设计
- `outline_draft` - 剧情大纲
- `outline_detail` - 章节细纲

### 数据处理

**角色管理：**
- 合并工作流角色和数据库角色
- 使用 `id: -1` 标记工作流角色
- 避免重复显示同名角色

**大纲编辑：**
- 分别查询大纲和细纲数据
- 支持多个卷的展开/收起
- 支持章节细纲的详细查看

**章节创作：**
- 根据章节号查找对应细纲
- 使用 `workflowApi.writeChapter()` 生成内容
- 支持自定义写作参数

## 使用说明

### 角色管理
1. 访问 `/novel/:novelId/characters`
2. 查看工作流生成的角色（带紫色星标）
3. 点击角色查看详细人设
4. 点击"保存到数据库"将角色持久化
5. 对已保存角色可点击编辑图标修改人设

### 大纲编辑
1. 访问 `/novel/:novelId/outline`
2. 查看工作流生成的剧情大纲（紫色区域）
3. 查看章节细纲（绿色区域）
4. 点击章节展开查看详细内容
5. 已保存的大纲在下方显示

### 继续创作
1. 访问 `/novel/:novelId/write`
2. 选择要创作的章节
3. 查看章节细纲（如有）
4. 配置写作设置（文风、对话占比等）
5. 点击"开始生成"
6. 等待生成完成查看内容

## 注意事项

1. 工作流数据必须在对应阶段完成后才能读取
2. 工作流角色需要手动保存到数据库才能持久化
3. 章节细纲在 `outline_detail` 阶段完成后可用
4. 生成章节内容可能需要较长时间，请耐心等待
